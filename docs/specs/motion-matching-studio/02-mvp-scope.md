# MotionMatching Studio MVP Scope

Created: 2026-05-14

## MVP Goal

The MVP should prove the full authoring loop without Unity:

```text
create/open workspace
create/select character
import Visual FBX
import animation FBX/BVH as clips
auto-map to visual skeleton
auto-suggest role/tags/foot contacts
preview selected clip on skinned mesh
build current Unity-compatible database files
enter Play Mode on a flat plane
run and jump with the current character
```

## Included In MVP

### Workspace

- File-system workspace.
- Multiple characters in a workspace.
- One active/current character for preview/play.
- `New` creates an empty workspace.
- Browser/Codex dev mode can use a browser/test workspace flow for early testing.
- If a browser workspace exists in browser/Codex dev mode, open it by default.
- Browser/Codex dev mode imports source assets by upload with a 50 MB MVP limit.
- Browser/Codex dev mode uses `.motionstudio/browser-workspace/`, with derived cache/logs kept out of portable project state.
- Operation logs are derived and excluded from portable zips by default.
- Empty workspace UI shows `Add Character` in the left panel and an empty 3D scene in the viewport.
- Portable JSON manifests.
- Imported source assets copied into workspace.
- Derived build/cache/log folders.
- Stable IDs plus human-readable folders.
- JSON property names use camelCase; serialized role/tag values use snake_case.

### Character

- Visual FBX import.
- Visual FBX is the required first setup step for a usable character.
- Add Character creates a character from a selected Visual FBX.
- New character name defaults from the Visual FBX filename.
- Replacing the Visual FBX does not rename the character.
- Visual FBX defines canonical skeleton.
- Import all skinned mesh parts bound to the canonical skeleton.
- Reject Visual FBX files with multiple independent character skeletons.
- Visual FBX provides the preview mesh and skin.
- Character-level tag vocabulary.
- Character-level preview tuning.
- Character-level contact detection defaults.
- Character-level validation summary.

### Clips

- One import action creates one clip.
- Source can be FBX or BVH.
- One selected take per import in MVP; single-take files select automatically, multi-take files show a take chooser.
- Reject animation source files with multiple independent animation skeletons.
- Clip folder name equals clip name.
- Unique clip names within a character.
- Clip source replacement.
- Clip include/exclude from build flag.
- Clip mirror-in-build augmentation flag.
- Imported clips default to included in build.
- Clip role combobox.
- Clip global tags as checkboxes.
- Per-clip contact detection override.
- Per-clip import settings for scale/axis/root-motion source with auto guesses and inspector overrides.
- Auto-suggest role and tags.
- Auto-detect and visualize foot contacts.
- Warnings/errors from validators.
- Validation status is visible with colors/icons in clip lists and inspectors.
- Left panel character/clip rows show aggregate validation badges and finding counts.
- Included clips with Error findings block build until fixed or excluded.
- Importing a clip immediately prepares it for preview by running mapping, validation, role/tag suggestions, and foot-contact detection.

### UI

- Vite + React + TypeScript frontend.
- Browser/dev mode usable inside Codex.
- Compact dark editor UI.
- Plain CSS/CSS variables/lucide icons; no heavy UI component library.
- Resizable splitters for left, right, and bottom panels; docking later.
- Left panel with a direct `Character -> clips` tree.
- Clip rows can show role chips, validation badges, and include/build actions through compact controls or context menus.
- Clip search and tag filters.
- Center Babylon.js 3D viewport.
- Right universal object inspector.
- Bottom timeline and logs/validation.
- Task list for background operations in the bottom logs area.
- Clickable validation findings that select the related entity and jump to an affected frame when available.
- Context menus for character rows, clip rows, viewport, and timeline.
- Confirm dialogs for destructive delete actions.
- Timeline frame/seconds ruler.
- Play/Pause and frame stepping.
- Importing an entity selects it and updates the inspector context.

### Viewport

- Load GLB preview mesh generated by backend from Visual FBX.
- Render skinned visual mesh.
- Perspective and orthographic camera modes.
- Orientation axes/cube helper.
- Viewport navigation with WASD, Shift speed, wheel zoom, middle-mouse pan, left-mouse orbit, frame selected, numpad view shortcuts, and useful context menus.
- Apply preview clip pose stream to the visual mesh.
- Material modes:
  - Textured
  - Flat
  - Shaded
- Debug toggles:
  - Show Skeleton
  - Show Foot Contacts
  - Show Root Motion Trail
  - Show Trajectory Query
  - Show Selected Pose Info
- Compact Play Mode debug HUD with state, current clip role/name, frame, speed, target speed, grounded/airborne, jump direction, selected pose, and warnings.

### Backend

- ASP.NET Core REST API.
- WebSocket progress/preview streams.
- Background task queue for import, reprocess, validation, and build operations.
- Import Visual FBX.
- Import animation FBX/BVH.
- Generate preview GLB/cache.
- Build workspace manifests.
- Run validators.
- Build Unity-compatible database files.
- Save build reports and operation logs.

### Build

- Built-in locomotion feature preset.
- Preset includes trajectory position/direction on simulation bone at frames `[20, 40, 60]`, plus left/right foot position/velocity and hips velocity pose features.
- Export current Unity-compatible format:
  - `.mmskeleton`
  - `.mmpose`
  - `.mmfeatures`
- Write derived output in workspace.
- Optional external export to Unity-compatible `MMDatabases` root.
- External database export overwrites existing generated database files atomically where practical.
- Build report includes exact feature layout/settings for manual Unity `MotionMatchingData` setup.
- Before full database output lands, the character inspector shows a first Build Plan report: included source clips, build-time mirrored entries, role coverage, skeleton coverage per included clip, foot contact present/missing status, and build readiness warnings/errors.
- `Generate Build Report` writes the current first build plan to `Builds/<CharacterName>/build-report.json`; this is a report artifact only, not the runtime database.
- Saved build reports remain viewable after browser reload through the character inspector.
- Saved build reports show `none/current/outdated` status based on a readiness fingerprint.
- `Generate Runtime Draft` writes `Builds/<CharacterName>/runtime-build-draft.json` plus first JSON drafts for `Builds/<CharacterName>/<CharacterName>.mmskeleton`, `.mmpose`, and `.mmfeatures`. It records the feature preset, skeleton bones/slots, planned pose samples per source/mirrored build entry, planned feature channels/samples, a user-chosen sample frame step, and a rough feature value preview from root-motion and foot-contact diagnostics. It exposes `none/current/outdated` runtime draft status in the character inspector, but does not claim Unity binary/database compatibility yet.
- Build runs in background.
- Dirty/outdated database warning strip in Play Mode.
- UI build is per character; workspace-level Build All and CLI build are not required for the MVP vertical slice.

### Play Mode

- Flat plane only.
- Current selected character only.
- WASD input.
- In Play Mode, WASD controls the character; in Animation Preview Mode, WASD controls the camera.
- Play Mode movement defaults to run; holding Shift switches to walk.
- Character movement can be constrained to a selected plane for 2.5D testing.
- Movement plane modes: Free3D, SideScrollerXY (Z locked), TopDownXZ (Y vertical/ground plane), Custom.
- Ground locomotion.
- Jump.
- Full role set expected for mechanics validation.
- Missing roles show warnings, but do not block Play Mode.

## Not Required In MVP

- Electron packaging.
- CLI build command.
- Workspace-level Build All.
- Portable zip/export command.
- Cloud/web hosted backend.
- Unity descriptor generation.
- Full manual feature editor.
- Full timeline range-tag editing.
- Full undo/redo.
- Blender dependency.
- High-fidelity material/render pipeline.
- Advanced retarget editor.
- Manual per-bone mapping override UI.
- Guided retargeting fix wizard.
- Multi-character Play Mode.
- Gamepad input.
- Clip recording/authoring from a placed capsule/controller timeline, including baking the recorded motion into a new clip.
- Obstacles, ledges, markers, or contextual interactions.
- Persistent build artifacts in portable zip.
- SQLite or database-backed workspace state.

## Suggested Implementation Order

When implementation is approved, start with:

1. JSON schemas and manifest/domain models.
2. Backend/frontend skeleton.
3. Empty workspace flow.
4. Left `Character -> clips` tree.
5. Empty Babylon.js viewport scene.

First coding milestone:

1. Open Studio in Codex browser.
2. Create/open the browser workspace.
3. Add Character by uploading the local Iyo Visual FBX.
4. Backend imports/copies the Visual FBX.
5. Backend generates preview GLB/cache.
6. Left tree shows the Iyo character.
7. Viewport shows the skinned Iyo mesh in bind/default pose.
8. Inspector shows character and validation summary.

Animation clips are intentionally deferred until after the first Visual FBX import/render milestone.

Initial toolchain:

- .NET 8.
- Node 22 LTS.
- npm.
- Vite + React + TypeScript.
- `MotionMatchingStudio.sln`.

## MVP Role Vocabulary

Each clip can have one `clipRole`:

- `idle_loop`
- `walk_loop`
- `run_loop`
- `turn_left`
- `turn_right`
- `turn_left_180`
- `turn_right_180`
- `jump`
- `fall_loop`
- `land`

Future role workflow:

- MVP ships a small built-in role vocabulary.
- Later, users should be able to add project/character-specific roles.
- Custom roles need authored metadata: display name, short description, optional grouping, and runtime/export behavior.
- Role vocabularies should be load/save/shareable JSON presets. A character can use the built-in vocabulary plus an imported or character-local custom vocabulary file.
- JSON presets should be portable between projects and should not contain machine-local paths.
- Built-in roles remain the default portable contract; custom roles are an authoring/runtime extension point, not tags.

## MVP Tag Vocabulary

Default global tags:

- `neutral`
- `calm`
- `happy`
- `angry`
- `sad`
- `tired`
- `injured`
- `combat`
- `stealth`
- `relaxed`

Tags describe style, mood, performance variant, or authoring context. Action categories such as walk, run, turn, jump, fall, and land belong in `clipRole`, not tags. Start/stop are transition phases/events, not active MVP roles. Users can add custom tags to the character vocabulary.

## MVP Validation

Validators report findings but do not silently override user decisions.

Minimum validators:

- Critical bone mapping validator.
- Bone length mismatch validator.
- Visual FBX import blocker validator.
- Animation source import blocker validator.
- Transform sanity validator for NaN/Infinity.
- Root/hips motion sanity validator.
- Required clip-role validator.
- Contact detection validator.
- Build compatibility validator for current Unity file format.

Severity levels:

- `Info`
- `Warning`
- `StrongWarning`
- `Error`

## MVP Success Criteria

The MVP is successful when:

1. A clean workspace can be created.
2. A Visual FBX can be imported and shown as a skinned mesh.
3. FBX/BVH clips can be imported as separate clips.
4. A clip can be previewed on the Visual FBX mesh.
5. Role/tags/contact suggestions appear automatically.
6. The user can edit clip role, tags, include/mirror flags, and contact settings.
7. A database build produces files compatible with the current Unity package naming/folder convention.
8. Play Mode runs on a flat plane inside the browser UI.
9. The selected character can run and jump using the built database, with visible warnings for missing/incomplete roles.
10. Project manifests and docs are sufficient for a fresh agent to continue work.
